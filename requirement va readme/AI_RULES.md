# CÁC QUY ƯỚC LÀM VIỆC DÀNH CHO AI (AI RULES)

File này dùng để lưu trữ các quy định, tiêu chuẩn, và cú pháp khi làm việc với AI trong dự án này. 
**Bất cứ khi nào có yêu cầu mới, hãy nhắc AI kiểm tra qua file này để tuân thủ luật nhé.**

---

## 1. Ngôn ngữ & Giao tiếp
* Bỏ qua bước tạo Kế hoạch (Implementation Plan) và xin phép. Cứ tự động triển khai code và thực thi ngay lập tức khi được giao việc.
* _(Ví dụ: Luôn trả lời ngắn gọn, trực tiếp vào vấn đề bằng tiếng Việt thay vì tiếng Anh.)_
* _(Ví dụ: Không tự ý xóa code cũ nếu chưa hỏi ý kiến.)_

## 2. Quy chuẩn về Code (Coding Standards)
* _(Ví dụ: Sử dụng Python với Flask, thư mục static chia theo css/ js/ img rõ ràng.)_
* _(Ví dụ: Đặt tên biến và hàm theo chuẩn `snake_case`, tên class là `CamelCase`.)_

## 3. Phong cách giao diện (UI/UX)
* _(Ví dụ: Sử dụng Vanilla CSS thay vì bất kỳ thư viện ngoài nào theo yêu cầu bắt buộc.)_
* _(Ví dụ: Mã màu chủ đạo: Xanh dương (#0056b3), Xanh lam nhạt (#e9ecef).)_

## 4. Cấu trúc Dự án
* _(Ví dụ: Mọi file Database cần nằm ở thư mục `instance/`)_
* _(Ví dụ: Không tạo trực tiếp file rác ra thư mục gốc, nếu cần nháp thì bỏ vào thư mục `scratch/`)_

## 5. Quản lý Source Code (Git)
* Từ giờ trở đi, chỉ thực hiện lệnh pull và push trên nhánh `master` của repository [https://github.com/sealdog17/SS2_InternshipFinder/tree/master](https://github.com/sealdog17/SS2_InternshipFinder/tree/master). Không tự ý checkout sang branch khác trừ phi được yêu cầu.
